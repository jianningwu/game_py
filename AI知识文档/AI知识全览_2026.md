# 三、大语言模型(LLM)技术详解

大语言模型(LLM)是当前AI革命的核心引擎。理解其技术原理，是深入掌握现代AI的基础。


## 3.1 整体架构

*(配套图表: chart_llm_arch.png -- LLM核心架构图)*

现代LLM(以GPT系列Decoder-Only架构为例)核心组件:

```
Input -> Token Embedding -> Position Encoding(RoPE) -> N x Decoder Layers -> Output -> Text
```

每层Decoder包含:
- **掩码多头自注意力**(Masked Multi-Head Self-Attention)
- **前馈网络**(FFN + SwiGLU激活)
- **RMSNorm层归一化 + 残差连接**


## 3.2 自注意力机制(Self-Attention)

*(配套图表: chart_attention.png -- 注意力机制原理图解)*

注意力机制是Transformer的核心创新: 处理每个词时，动态决定应该关注上下文中的哪些词。

**核心公式:**

```
Attention(Q, K, V) = softmax(Q*K^T / sqrt(d_k)) * V
```

| 向量 | 全称 | 含义 |
| --- | --- | --- |
| **Q** | Query(查询) | 当前token想要查询的信息 -- 我在找什么? |
| **K** | Key(键) | 上下文token的索引标签 -- 我是什么? |
| **V** | Value(值) | 上下文token的实际内容 -- 我包含什么信息? |


**为什么要除以 sqrt(d_k)?**

当键向量维度d_k较大时，点积QK^T的方差增大，导致softmax进入梯度饱和区(梯度消失)。
除以sqrt(d_k)可保持方差稳定 -- 省略此操作，训练失败率将提高60%以上。


## 3.3 多头注意力(Multi-Head Attention)

单头注意力只能关注一种语义关系。多头注意力并行运行多个注意力头，
每个头专注不同语言现象: 语法结构、实体关系、指代消解等。
典型LLM使用32-128个注意力头，最后拼接输出并通过线性变换融合。


## 3.4 三大关键技术


### 旋转位置编码(RoPE)

- **问题:** 自注意力本身不感知词序，需人工注入位置信息
- **原理:** 对Q和K向量施加旋转操作，将相对位置隐式编码到注意力计算中
- **优势:** 支持长度外推 -- 16K训练的模型扩展到32K推理，性能下降不到5%
- **采用者:** LLaMA、Qwen、DeepSeek、Mistral等主流模型标配


### 分组查询注意力(GQA)

- **问题:** 标准MHA的K/V投影消耗大量显存
- **原理:** 注意力头分若干组，组内共享K/V投影
- **效果:** K/V显存降至原来的1/G，模型容量不受影响
- **极端情况:** G=1为MQA(多查询注意力)，G=H为标准MHA


### 混合专家模型(MoE)

- **核心思想:** 每次推理只激活部分专家，而非全部参数
- **效果:** GLM-4.7 3550亿总参 -> 每次仅激活320亿
- **意义:** 更大知识容量但同等计算成本 -- 更聪明但不更费电
- **2026年地位:** 高效大模型的标配架构

| **连续批处理** | 动态合并多请求 | 吞吐量10倍+ |

## 4.3 AI - ML - DL - LLM 的关系

四者的关系可以理解为同心圆结构:

```
+--------------------------------------------+
|              人工智能 (AI)                 |
|   +------------------------------------+   |
|   |         机器学习 (ML)              |   |
|   |   +----------------------------+   |   |
|   |   |      深度学习 (DL)        |   |   |
|   |   |   +--------------------+  |   |   |
|   |   |   |  Transformer/LLM   |  |   |   |
|   |   |   |  AI Agent          |  |   |   |
|   |   |   +--------------------+  |   |   |
|   |   +----------------------------+   |   |
|   +------------------------------------+   |
+--------------------------------------------+
```

- **AI** 最大: 任何让机器表现出智能行为的技术
- **ML** 是AI的实现方式: 让机器从数据中学习而非手工编码规则
- **DL** 是ML的一种: 使用深层神经网络自动提取特征
- **Transformer/LLM** 是DL的前沿: 处理序列数据，实现语言理解生成

# 五、AI Agent 智能体详解

AI Agent(智能体)是2025-2026年AI领域最重要的范式转变。
传统LLM是知识渊博的顾问，AI Agent则是能自主完成任务的数字员工。
它能感知环境、制定计划、调用工具、执行动作，并根据反馈自我修正。


## 5.1 Agent 核心架构

*(配套图表: chart_agent_arch.png -- AI Agent架构图)*

AI Agent由六大核心模块组成:

```
                       +--------------+
                       |   大语言模型   |
                       |  (推理/决策)  |
                       +-------+------+
                               |
       +-----------+-----------+-----------+
       |           |           |           |
       v           v           v           v
  +--------+ +--------+ +--------+ +--------+
  | 感知模块| | 任务规划| | 记忆系统| | 工具调用|
  |文本/图像| | 目标分解| |短期/长期| |代码/API |
  |语音/API | | 步骤排序| | RAG检索 | |数据库查询|
  +--------+ +--------+ +--------+ +--------+
                               |
                               v
                        +------------+
                        | 执行与反馈 |
                        | 验证/修正  |
                        +------------+
```


## 5.2 Agent 四大核心能力

| 能力 | 说明 | 示例 |
| --- | --- | --- |
| **感知(Perception)** | 多模态输入理解环境和用户意图 | 读文档/分析图片/监听API |
| **规划(Planning)** | 将复杂目标分解为可执行子任务 | 旅行规划->机票->酒店->行程 |
| **记忆(Memory)** | 短期(上下文)+长期(向量库/知识图谱) | 记住偏好/项目背景/对话历史 |
| **行动(Action)** | 工具调用和代码执行改变外部世界 | 发邮件/操作文件/查数据库/运行代码 |


## 5.3 单智能体 vs. 多智能体

| 类型 | 架构 | 适用场景 |
