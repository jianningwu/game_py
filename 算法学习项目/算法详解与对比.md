## 一、复杂度分析基础

### 大 O 表示法

```
O 表示法描述算法随输入规模增长的变化趋势（忽略常数和低阶项）。

常见复杂度从快到慢:
O(1)    常数时间     - 数组随机访问、哈希表查找
O(log n) 对数时间     - 二分查找、平衡树操作
O(n)    线性时间     - 遍历数组、线性搜索
O(n log n) 线性对数  - 归并排序、快速排序(平均)、堆排序
O(n^2)  平方时间     - 冒泡排序、选择排序、插入排序
O(2^n)  指数时间     - 递归求斐波那契(无记忆化)、子集枚举
O(n!)   阶乘时间     - 旅行商问题暴力解、全排列

直观感受 (n=100时):
O(1):     1 次
O(log n): ~7 次
O(n):     100 次
O(n log n): ~700 次
O(n^2):   10,000 次
O(2^n):   天文数字!
```

### 主定理 (Master Theorem)

```
用于快速分析分治算法的复杂度:
T(n) = a*T(n/b) + f(n)

情况1: f(n) = O(n^{log_b(a) - e})  -> T(n) = O(n^{log_b(a)})
情况2: f(n) = O(n^{log_b(a)})      -> T(n) = O(n^{log_b(a)} * log n)
情况3: f(n) = O(n^{log_b(a) + e})  -> T(n) = O(f(n))

归并排序: T(n) = 2T(n/2) + O(n)  -> a=2, b=2, f(n)=O(n)
  log_2(2)=1, f(n)=O(n^1) -> 情况2, O(n log n)
```

## 二、排序算法

### 2.1 冒泡排序 (Bubble Sort)

```
过程: [5, 3, 8, 1] 从小到大
第1轮: [3, 5, 1, 8]  比较相邻，大的右移，最大的 8 沉底
第2轮: [3, 1, 5, 8]  5 沉底
第3轮: [1, 3, 5, 8]  完成

特点:
  时间: O(n^2)  空间: O(1)  稳定: 是
  每轮把最大的元素"冒泡"到最后
  优化: 如果一轮没有交换，说明已有序可提前退出
```

### 2.2 选择排序 (Selection Sort)

```
过程: [5, 3, 8, 1]
第1轮: [1, 3, 8, 5]  找[5,3,8,1]中最小=1，与第1位交换
第2轮: [1, 3, 8, 5]  找[3,8,5]中最小=3，已在位置
第3轮: [1, 3, 5, 8]  找[8,5]中最小=5，与第3位交换

特点:
  时间: O(n^2)  空间: O(1)  稳定: 否
  每轮找未排序部分的最小值，放到已排序末尾
  交换次数最少: 最多 n-1 次交换
```

### 2.3 插入排序 (Insertion Sort)

```
过程: [5, 3, 8, 1]
第1轮: [5 | 3, 8, 1]  拿 3，比 5 小，5 右移，3 插入
第2轮: [3, 5 | 8, 1]  拿 8，比 5 大，不动
第3轮: [3, 5, 8 | 1]  拿 1，比 8/5/3 都小，全部右移，1 插入最前
      [1, 3, 5, 8]

特点:
  时间: O(n^2)  空间: O(1)  稳定: 是
  类似打扑克牌时整理手牌
  对小规模数据(<=50)和"基本有序"的数据非常快 O(n)
  Timsort (Python 内置排序) 的组成部分
```

### 2.4 归并排序 (Merge Sort)

```
过程: [5, 3, 8, 1, 6, 2]
        分
    [5,3,8]    [1,6,2]
    分          分
  [5][3,8]   [1][6,2]
  合          合
  [3,5,8]    [1,2,6]
        合
    [1,2,3,5,6,8]

特点:
  时间: O(n log n) (严格，不依赖数据分布)
  空间: O(n) (需要额外数组)
  稳定: 是
  适合: 链表排序、外部排序(数据太大无法一次载入内存)
```

### 2.5 快速排序 (Quick Sort)

```
过程: [5, 3, 8, 1, 6]
选 pivot=5
分区: [3, 1]  [5]  [8, 6]    左边<5, 右边>5
      递归排左右

特点:
  时间: 平均 O(n log n), 最坏 O(n^2) (每次 pivot 都是极值)
  空间: O(log n) (递归栈)
  稳定: 否
  实际最快(常数因子小，缓存友好)
  优化: 三数取中选 pivot, 小数组切换插入排序
```

### 2.6 堆排序 (Heap Sort)

```
过程:
1. 建堆: 将数组转为最大堆 O(n)
2. 排序: 反复取堆顶(最大值)放到末尾，调整堆 O(n log n)

特点:
  时间: O(n log n) (严格)
  空间: O(1) (原地排序，归并做不到)
  稳定: 否
  适合: 内存极度受限且需要 O(n log n) 的场景
```

### 2.7 计数排序 / 基数排序

```
计数排序: 统计每个值出现次数，适合整数、范围小的场景
  时间: O(n+k)  k=数据范围  空间: O(k)  稳定: 是

基数排序: 按位排序(个位->十位->百位)，适合定长数据
  时间: O(d*(n+k))  d=位数  空间: O(n+k)  稳定: 是
```

### 排序算法总对比

| 算法 | 平均时间 | 最坏时间 | 空间 | 稳定 | 适合场景 |
|------|---------|---------|------|------|---------|
| 冒泡 | O(n^2) | O(n^2) | O(1) | 是 | 教学用 |
| 选择 | O(n^2) | O(n^2) | O(1) | 否 | 交换次数最少 |
| 插入 | O(n^2) | O(n^2) | O(1) | 是 | 小规模/基本有序 |
| 归并 | O(n log n) | O(n log n) | O(n) | 是 | 链表/外部排序 |
| 快速 | O(n log n) | O(n^2) | O(log n) | 否 | **通用首选** |
| 堆 | O(n log n) | O(n log n) | O(1) | 否 | 内存受限 |
| 计数 | O(n+k) | O(n+k) | O(k) | 是 | 整数/小范围 |
| 基数 | O(d*(n+k)) | O(d*(n+k)) | O(n+k) | 是 | 定长字符串 |
| Timsort | O(n log n) | O(n log n) | O(n) | 是 | Python/Java 内置 |

## 三、搜索算法

### 3.1 二分查找与变体

```
经典二分查找: 在有序数组中找目标值 O(log n)

变体（面试常考）:
  1. 找第一个等于 target 的位置 (左边界)
  2. 找最后一个等于 target 的位置 (右边界)
  3. 找第一个 >= target 的位置 (lower_bound)
  4. 找第一个 > target 的位置 (upper_bound)
  5. 在旋转排序数组中查找

二分查找模板 (左闭右闭):
  left, right = 0, len(arr)-1
  while left <= right:
      mid = left + (right - left) // 2   # 防溢出
      if arr[mid] == target: return mid
      elif arr[mid] < target: left = mid + 1
      else: right = mid - 1
  return -1
```

### 3.2 DFS vs BFS 对比

```
DFS (深度优先, 栈/递归):          BFS (广度优先, 队列):
       ①                              ①
      / \                            / \
     ②   ③    DFS: ①->②->④->⑤->③    ②   ③    BFS: ①->②->③->④->⑤
    / \                             / \
   ④  ⑤                            ④  ⑤

DFS 用栈, 走到底回溯             BFS 用队列, 一层层扩散
适合: 全部路径、连通性           适合: 最短路径(无权)、层序遍历
空间: O(h) 树高                空间: O(w) 最大宽度
```

## 四、递归与回溯 (Backtracking)

### 核心思想

```
回溯 = 递归 + 状态恢复
"一条路走到黑，走不通就退回一步"

回溯三要素:
  1. 路径: 已经做出的选择
  2. 选择列表: 当前可做的选择
  3. 结束条件: 到达决策树底层

模板:
  def backtrack(路径, 选择列表):
      if 满足结束条件:
          result.add(路径)
          return
      for 选择 in 选择列表:
          做选择
          backtrack(路径, 新的选择列表)
          撤销选择  (关键!)
```

### 经典例题

| 问题 | 描述 | 复杂度 |
|------|------|--------|
| 全排列 | 给定数组，返回所有排列 | O(n!) |
| N 皇后 | N×N 棋盘放 N 个皇后互不攻击 | O(n!) |
| 子集 | 返回数组的所有子集 | O(2^n) |
| 组合总和 | 找数组中和为 target 的组合 | O(2^n) |
| 数独求解 | 9×9 数独填数字 | O(9^m), m=空格数 |

## 五、分治法 (Divide & Conquer)

### 三步走

```
1. 分解(Divide): 将原问题分解为若干规模较小的子问题
2. 解决(Conquer): 递归求解子问题(子问题足够小时直接求解)
3. 合并(Combine): 将子问题的解合并为原问题的解

经典例子:
  归并排序: 分->左半右半, 治->递归排序, 合->合并两个有序数组
  快速排序: 分->分区, 治->递归排左右
  二分查找: 分->取中点, 治->递归一半
```

## 六、动态规划 (Dynamic Programming)

### 核心思想

```
DP = 记忆化搜索 + 最优子结构

三个核心概念:
  1. 状态定义: dp[i] 代表什么? (最难的一步)
  2. 状态转移方程: dp[i] 和 dp[i-1] 等的关系?
  3. 初始值/边界: dp[0] = ?

与分治法的区别: DP 子问题有重叠(会重复计算), 分治子问题独立

与贪心的区别: DP 全局最优, 贪心局部最优(不一定全局最优)
```

### 经典 DP 模型

| 模型 | 描述 | 转移方程 | 复杂度 |
|------|------|---------|--------|
| **斐波那契** | 最优子结构入门 | `dp[i]=dp[i-1]+dp[i-2]` | O(n) |
| **01 背包** | 选/不选，容量限制 | `dp[i][w]=max(dp[i-1][w], dp[i-1][w-w_i]+v_i)` | O(nW) |
| **完全背包** | 每件物品无限次 | `dp[i][w]=max(dp[i-1][w], dp[i][w-w_i]+v_i)` | O(nW) |
| **最长公共子序列(LCS)** | 两个字符串的公共子序列 | `if a[i]==b[j]: dp+1 else: max(dp[i-1][j],dp[i][j-1])` | O(mn) |
| **最长递增子序列(LIS)** | 子序列严格递增 | `dp[i]=max(dp[j])+1 (j<i, a[j]<a[i])` | O(n^2) / O(n log n) |
| **编辑距离** | 字符串变换最少操作数 | `dp[i][j]=min(dp[i-1][j]+1, dp[i][j-1]+1, dp[i-1][j-1]+cost)` | O(mn) |
| **矩阵链乘法** | 最优加括号方式 | 区间 DP 模板 | O(n^3) |
| **打家劫舍** | 不能偷相邻 | `dp[i]=max(dp[i-1], dp[i-2]+nums[i])` | O(n) |

### 解题套路

```
1. 先尝试暴力递归 (明确状态和选择)
2. 画出递归树，观察是否有重叠子问题
3. 加备忘录 -> 自顶向下 DP (记忆化搜索)
4. 改迭代 -> 自底向上 DP (数组递推)
5. 优化空间 (滚动数组)

识别 DP 的信号:
  - 求最值 (最大/最小/最多/最少)
  - 求方案数 (有多少种方法)
  - 求可行性 (能不能达到)
  - 问题有明显的最优子结构
```

## 七、贪心算法 (Greedy)

### 核心思想

```
每步都选当前看起来最好的选择，不回头。
"鼠目寸光" —— 只有问题具备贪心选择性质时才能得到全局最优。

贪心 vs DP:
  贪心: 每步选最优 -> 可能不是全局最优
  DP:   考虑所有可能 -> 保证全局最优

贪心成立的前提:
  1. 贪心选择性质: 局部最优能推出全局最优
  2. 最优子结构: 问题的最优解包含子问题的最优解
```

### 经典例题

| 问题 | 贪心策略 |
|------|---------|
| **找零钱** | 每次找面额最大的（硬币面额特殊时成立） |
| **活动选择** | 每次选结束时间最早的 |
| **哈夫曼编码** | 每次合并频率最小的两个节点 |
| **最小生成树 Prim/Kruskal** | 每次选最小权值边 |
| **Dijkstra 最短路径** | 每次选距离最小的节点扩展 |
| **跳跃游戏** | 维护最远能到达的位置 |

## 八、图算法

### 最短路径

| 算法 | 适用图 | 时间复杂度 | 核心思想 |
|------|--------|-----------|---------|
| **BFS** | 无权图 | O(V+E) | 队列，层层扩散 |
| **Dijkstra** | 非负权 | O((V+E)logV) | 贪心+优先队列，每次选最近节点 |
| **Bellman-Ford** | 可负权 | O(VE) | 松弛所有边 V-1 轮 |
| **Floyd-Warshall** | 全源 | O(V^3) | `dp[k][i][j]` 节点中转 |
| **A\*** | 有权图 | O(E) | Dijkstra + 启发式估计 |

### 最小生成树

| 算法 | 思想 | 时间复杂度 |
|------|------|-----------|
| **Prim** | 点扩展：每次选与已选集合最近的节点 | O((V+E)logV) |
| **Kruskal** | 边合并：每次选最小的不形成环的边 | O(E log E) (并查集) |

### 拓扑排序

```
适用: 有向无环图(DAG)
算法: 每次选入度为 0 的节点，移除其所有边
应用: 课程安排、构建依赖、任务调度
时间: O(V+E)
```

## 九、字符串算法

### 字符串匹配

```
问题: 在文本串 T 中找模式串 P 的所有出现位置

朴素算法 O(n*m):
  滑动比较每个位置

KMP O(n+m):
  核心: 当匹配失败时，利用已匹配的信息，不回退文本指针
  关键: next 数组 (PMT 部分匹配表)
    next[i] = P[0:i] 的最长相等前后缀长度

    例如 P = "ABABC":
    P:    A  B  A  B  C
    next: 0  0  1  2  0
    含义: 当 P[4]='C' 匹配失败时，已匹配 "ABAB",
          最长前后缀 "AB", 直接跳到 P[2] 继续

Boyer-Moore O(n/m) 平均:
  从模式串末尾往前匹配，利用坏字符和好后缀跳过大段文本

Rabin-Karp O(n+m) 平均:
  用滚动哈希比较，哈希值相同再逐字符确认
```

## 十、滑动窗口 (Sliding Window)

### 核心思想

```
维护一个窗口 [left, right]，右扩时更新状态，满足条件时左缩。
将 O(n^2) 的暴力枚举优化为 O(n)。

模板:
  left = 0
  for right in range(n):
      将 arr[right] 加入窗口
      while 窗口不满足条件:
          将 arr[left] 移出窗口
          left += 1
      更新答案

经典题:
  无重复字符的最长子串
  最小覆盖子串
  长度最小的子数组(和 >= target)
  至多包含 K 个不同字符的最长子串
```

## 十二、位运算技巧

```
常用操作:
  x & 1      判断奇偶 (1=奇, 0=偶)
  x >> 1     除以 2
  x << 1     乘以 2
  x & (x-1)  去掉最低位的 1 (统计 1 的个数)
  x & -x     只保留最低位的 1 (树状数组)
  x ^ x = 0  自己异或自己 = 0
  x ^ 0 = x  任何数异或 0 = 自己

应用:
  只出现一次的数字: 所有数异或，结果就是只出现一次的数
  快速幂: 用位运算拆分指数 O(log n)
  子集枚举: for mask in range(1<<n)
  状态压缩 DP: 用整数的 bit 表示集合状态
```

```
需要解决什么问题?
│
├── 数据需要排序? ───> 排序算法

│   ├── 稳定 ───> 归并排序
│   ├── 内存小 ───> 堆排序
│   ├── 基本有序 ───> 插入排序
│   └── 数据范围小 ───> 计数排序
│

│   ├── 有序数组找值 ───> 二分查找 O(log n)
│   ├── 迷宫/所有路径 ───> DFS(回溯)
│   └── 最短路径(无权) ───> BFS
│
├── 求最/最优解?
│   ├── 暴力可解 + 有重叠子问题 ───> 动态规划
│   ├── 局部最优=全局最优 ───> 贪心
```
