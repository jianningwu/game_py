# 飞机大战 EX — 设计文档

## 1. 项目概述

飞机大战 EX 是一款基于 Python Pygame 开发的纵向卷轴射击游戏（STG，俗称"打飞机"）。玩家操控战机在星空中迎击敌机，通过切换不同弹药应对战局，在敌机愈发密集的攻势中争取高分。

**核心文件：** `plane_war_ex.py`（约 1240 行，单文件实现）
**目标平台：** Windows 桌面（打包为独立 `.exe`，双击即玩）

## 3. 软件架构

### 3.1 整体架构

```
┌─────────────────────────────────────────┐
│               Game (主控制器)             │
│  ┌────────┐ ┌────────┐ ┌──────────────┐ │
│  │ handle │ │ update │ │    draw      │ │
│  │_events │ │        │ │              │ │
│  └────────┘ └────────┘ └──────────────┘ │
│         │          │          │          │
│         ▼          ▼          ▼          │
│  键盘事件   游戏逻辑更新   画面渲染        │
│  菜单导航   物理/碰撞/  精灵绘制/特效/    │
│  角色选择   计分/生成     HUD 叠加       │
└─────────────────────────────────────────┘
```

游戏遵循经典的**游戏循环架构**，每帧（约 16.7ms，60 FPS）执行三个步骤：

```
while running:
    handle_events()   # 处理用户输入（键盘按键）
    update()          # 更新所有游戏对象状态
    draw()            # 绘制所有画面到窗口
    clock.tick(FPS)   # 控制帧率
```

### 3.2 游戏状态机

游戏包含 4 个状态，状态转换清晰：

```
┌──────────┐   空格/Enter    ┌──────────┐
│  MENU    │ ──────────────> │  SELECT  │
│ (主菜单)  │                 │ (角色选择) │
└──────────┘                 └────┬─────┘
      ▲                          │ 空格/Enter
      │                          ▼
      │                   ┌──────────┐
      │   空格/Enter      │ PLAYING  │
      │ <─────────────── │  (游戏中)  │
      │                   └────┬─────┘
      │                        │ 生命归零
      │                        ▼
      │                   ┌──────────┐
      └───────────────────│GAME_OVER │
                          │ (游戏结束)│
                          └──────────┘
```

- **MENU**: 显示标题和简要操作说明，列出 4 个角色预览
- **SELECT**: 左右键切换角色，显示角色详细属性
- **PLAYING**: 核心游戏进行中
- **GAME_OVER**: 显示最终得分，按空格返回主菜单

## 4. 类结构详解

### 4.1 类继承关系

```
pygame.sprite.Sprite
  ├── Player          (玩家飞机)
  ├── Bullet          (玩家子弹)
  ├── EnemyBullet     (敌机子弹)
  ├── Enemy           (敌机基类)
  │   ├── SmallEnemy  (小型敌机)
  │   ├── NormalEnemy (普通敌机)
  │   ├── EliteEnemy  (精英敌机)
  │   └── BossEnemy   (Boss敌机)
  └── AmmoPickup      (弹药掉落道具)

独立类（非 Sprite）:
  ├── Particle        (单个粒子)
  ├── ExplosionEffect (爆炸特效组合)
  └── Background      (背景系统)
```

### 4.2 核心类职责

| 类 | 职责 | 关键属性 |
|------|----------|----------|
| `Game` | 主控制器：窗口、循环、碰撞、计分、状态管理 | `state`, `score`, `level`, `combo` |
| `Player` | 玩家输入响应、射击、伤害、角色属性 | `lives`, `ammo_type`, `power` |
| `Bullet` | 按弹药类型生成不同外观/伤害的子弹 | `damage`, `aoe`, `ammo_type` |
| `Enemy` (基类) | 敌机通用行为：移动、射击冷却、受伤 | `hp`, `speed`, `score`, `shoot_chance` |
| `SmallEnemy` | 低血量高速，不射击 | hp=1, speed=2+ |
| `NormalEnemy` | 中血量，会射击 | hp=2+, `shoot_chance=0.02+` |
| `EliteEnemy` | 高血量，左右移动射击 | hp=5+, 横向运动 |
| `BossEnemy` | 超高血量，入场动画，三方向弹幕 | hp=15+, 三发子弹 |
| `ExplosionEffect` | 粒子爆炸系统 | `particles[]`, `flash`, `shock` |
| `Background` | 多层背景渲染 | `stars[]`, `nebulae[]`, `buildings[]` |

## 5. 游戏机制

### 5.1 角色系统

4 个可选角色，各具特色：

| 角色 | 生命 | 速度 | 射速倍率 | 特殊能力 |
|------|------|------|----------|----------|
| 普通飞行员 | 3 | 6 | 1.0x | 无 |
| 雷霆战机 | 5 | 7 | 0.85x | 初始双连弹 |
| **无限火力** | 3 | 6 | **0.45x** | **全程满弹药+额外子弹** |
| **不死战神** | **∞** | 6 | 1.0x | **不死之身** |

属性通过 `CHARACTERS` 字典数组配置，`Player.__init__` 根据 `char_index` 读取：

```python
CHARACTERS = [
    { "name": "无限火力", "fire_rate": 0.45, "infinite_power": True, ... },
    { "name": "不死战神", "lives": 999, "infinite_lives": True, ... },
]
```

### 5.2 弹药系统

5 种弹药类型，战斗中通过击败敌机掉落拾取（按 Q 键切换）：

| 弹药 | 伤害 | 射速(ms) | 弹道 | 特殊 |
|------|------|----------|------|------|
| 普通弹 N | 1 | 160 | 单发直线 | - |
| 双连弹 D | 1 | 150 | 双发平行 | - |
| 散弹 S | 1 | 180 | 3 方向 | 伤害覆盖范围广 |
| 激光 L | **2** | **100** | 细长高穿透 | **高速高伤** |
| 导弹 M | **5** | 400 | 双发 | **范围 AOE 50px** |

弹药数据定义在 `AMMO_DATA` 字典，每个弹药类型包含伤害、延迟、弹道数量、扩散角度。导弹还带 `aoe` 属性，击中后对周围 50 像素内的敌机造成溅射伤害。

### 5.3 敌机系统

敌机共 4 种，难度随等级 `level` 动态提升：

```
level = enemies_killed // 15 + 1
```

| 敌机 | 生命 | 速度 | 射击 | 分数 | 特殊行为 |
|------|------|------|------|------|----------|
| 小型 | 1 | 2+0.2L | 否 | 100 | 直线下坠 |
| 普通 | 2+L/5 | 1.5+0.15L | 概率 | 200 | 直线，概率射击 |
| 精英 | 5+L/3 | 1.2+0.1L | 概率 | 500 | **左右移动**，等级≥5出现 |
| Boss | 15+3L | 0.8 | **三方向** | 2000 | **入场动画**，每 40 只出现一次 |

- 生成间隔随等级缩短：`max(12, 55 - level * 3)`
- 敌机速度、生命、射击概率均随等级线性增长
- Boss 有入场动画（从顶部飞到设定位置停住），随后进行三方向弹幕攻击

### 5.4 计分与连击

```
单次击落得分 = 敌机基础分数 × (1 + combo // 10)
```

- 每连续击落一架敌机，combo +1
- 每 10 连击提供 1 倍额外分数加成
- 连击计时器 2 秒（120帧）内未击落新敌机，连击归零

### 5.5 无限火力角色机制

"无限火力"角色拥有：
1. **射速加成**：`fire_rate = 0.45`，所有弹药射速减半
2. **额外子弹**：`shoot()` 方法中检测 `infinite_power` 标志，在标准射击基础上额外添加两侧的偏移子弹
3. **弹药不受限**：始终可以使用满弹药效果

### 5.6 不死战神角色机制

"不死战神"角色设置 `infinite_lives = True`：
1. **生命 999**：实际上永远不会耗尽的数值
2. **受伤处理**：`hit()` 方法中检测 `infinite_lives`，受伤后仅短暂无敌 30 帧，不扣血、不触发游戏结束

## 6. 图形与特效

### 6.1 背景系统 (Background 类)

所有图形均通过 Pygame 绘制函数生成，**无任何外部图片资源**。

背景由 6 层叠加渲染：

```
第1层: 渐变天空 ── 逐像素垂直渐变（深蓝→蓝紫→紫黑）
第2层: 星云团 ──── 6 个半透明彩色圆形，缓慢漂移
第3层: 闪烁星辰 ── 120 颗星星，正弦波呼吸闪烁
第4层: 城市剪影 ── 随机生成的建筑群，窗户随机亮灯
第5层: 远山轮廓 ── 3 层多边形山脉
第6层: 底部雾气 ── 从下往上的半透明黑色渐变
```

**闪烁算法：** 每颗星星的亮度按正弦波变化：
```python
twinkle = math.sin(twinkle_phase) * 0.3 + 0.7
brightness = base_bright * twinkle
```

### 6.2 爆炸特效系统

爆炸特效由三个子效果组合：

1. **粒子群**：60 个（普通 25 个）粒子向各个方向飞散，带重力、空气阻力和透明度衰减
2. **中心闪光**：3 层同心圆，从白到透明逐层扩散
3. **冲击波**：白色圆环向外扩张，透明度递减

```
每个粒子属性: 位置(x,y) | 速度(vx,vy) | 颜色 | 大小 | 寿命 | 重力
粒子更新: 位置+=速度, 速度*=0.98(阻力), 寿命递减
粒子绘制: 直径和透明度随寿命衰减
```

Boss 爆炸额外生成 30 个金色火花粒子，并触发**屏幕震动**。

### 6.3 屏幕震动

```python
shake_amount = 12  # Boss 爆炸时触发
# 每帧衰减
shake_amount *= 0.85
if shake_amount < 0.5: shake_amount = 0
# 绘制时应用随机偏移
offset_x = random(-shake_amount, shake_amount)
offset_y = random(-shake_amount, shake_amount)
```

### 6.4 玩家飞机绘制

使用 Pygame 多边形绘制，逐顶点构建飞机形状：
- 10 个顶点的绿色/青色等箭头形机身
- 高光面（颜色+60）
- 蓝色圆形座舱 + 高光点
- 橙色引擎尾焰

## 7. 碰撞检测

使用 Pygame Sprite 提供的碰撞函数：

| 碰撞对 | 方法 | 回调行为 |
|--------|------|----------|
| 子弹 → 敌机 | `groupcollide(bullets, enemies, True, False)` | 子弹消失，敌机扣血 |
| 敌机子弹 → 玩家 | `spritecollide(player, enemy_bullets, True)` | 玩家扣血/无敌 |
| 敌机本体 → 玩家 | `spritecollide(player, enemies, False)` | 敌机销毁，玩家扣血 |
| 玩家 → 道具 | `spritecollide(player, powerups, True)` | 拾取，切换弹药 |

注意：`groupcollide` 的 dokill 参数为 `(True, False)`，即子弹命中后销毁，但敌机由 `hit()` 方法根据剩余 HP 决定是否调用 `kill()`。

## 8. Windows 打包部署

### 8.1 使用 PyInstaller 打包为独立 exe

```bash
# 安装依赖
pip install pygame pyinstaller

# 打包为单文件无控制台窗口的 exe
pyinstaller --onefile --windowed --name "飞机大战EX" plane_war_ex.py
```

**参数说明：**

| 参数 | 作用 |
|------|------|
| `--onefile` | 打包为单个 exe 文件 |
| `--windowed` | 不显示命令行控制台窗口 |
| `--name` | 指定输出 exe 文件名 |

### 8.2 输出结构

```
plane_war_ex/
├── plane_war_ex.py       # 源代码（约 1240 行）
├── 设计文档.md            # 本文件
├── dist/
│   └── 飞机大战EX.exe     # 打包后的可执行文件（约 27MB）
└── build/                # 构建临时文件（可删除）
```

### 8.3 运行方式

**方式一（推荐）：** 双击 `dist/飞机大战EX.exe`

**方式二（开发调试）：**
```bash
python plane_war_ex.py
```

### 8.4 为什么 exe 体积 27MB

Pygame 本身依赖 SDL2、SDL_image、SDL_mixer、SDL_ttf 等多个底层库，加上 Python 解释器和标准库，PyInstaller 会完整打包所有依赖。这是正常的，不影响运行速度。

## 10. 扩展开发建议

### 10.1 增加音效
当前 `_init_sounds()` 使用空音频占位。替换为真实音效：
```python
self.shoot_sound = pygame.mixer.Sound("shoot.wav")
self.explosion_sound = pygame.mixer.Sound("explosion.wav")
```
在射击和爆炸处调用 `.play()`。

### 10.2 增加更多角色/弹药
- 在 `CHARACTERS` 列表中添加新角色字典
- 在 `AMMO_DATA` 字典中定义新弹药属性
- 在 `Bullet.__init__` 和 `Player.shoot` 中添加绘制和发射逻辑

### 10.3 增加关卡/Boss 战
当前 Boss 每 40 个敌机出现一次。可以在特定分数或时间触发 Boss，增加 Boss 的弹幕模式（扇形弹、追踪弹等）。

### 10.4 图片资源替换
当前所有图形使用 Pygame draw 函数绘制。如需替换为图片资源，在 `_create_surface` 方法中改为：
```python
self.image = pygame.image.load("player.png").convert_alpha()
```

## 11. 常见问题

**Q: 双击 exe 没反应？**
A: 打开命令行运行 `dist/飞机大战EX.exe` 查看错误输出。常见原因：缺少 VC++ 运行库。

**Q: 如何修改窗口大小？**
A: 修改 `plane_war_ex.py` 顶部的 `SCREEN_WIDTH` 和 `SCREEN_HEIGHT` 常量，然后重新打包。

**Q: 游戏卡顿？**
A: 检查是否有其他程序占用 GPU。本游戏使用 CPU 渲染，60 FPS 需要 2.0GHz+ 的 CPU。
