# 通用 C 数据结构库 — 设计文档

## 1. 项目概述

本项目提供一个高质量、可复用、符合商业标准的 C 语言数据结构库，包含 6 种常用数据结构：

| 数据结构 | 文件 | 核心特性 |
|----------|------|----------|
| 动态数组 | `darray.h/c` | 泛型、自动扩容、随机访问 O(1) |
| 双向链表 | `dlist.h/c` | 哨兵节点环形实现、迭代器 |
| 消息队列 | `mqueue.h/c` | 线程安全、多生产者/多消费者、超时机制 |
| 哈希表 | `htable.h/c` | 链地址法、自动 rehash、自定义哈希 |
| AVL 树 | `btree.h/c` | 严格平衡 O(log n)、4 种遍历、范围查询 |
| 图 | `graph.h/c` | 邻接表、BFS/DFS、Dijkstra、拓扑排序 |

### 设计目标

- **通用性**：所有结构通过 `void*` + 元素大小实现泛型，可存储任意类型
- **健壮性**：所有函数返回错误码，无隐藏的 `abort()` 或 `exit()`
- **内存安全**：无内存泄漏，提供完整的 destroy/clear 接口
- **可移植性**：ANSI C99，消息队列通过预处理宏支持 Windows（CRITICAL_SECTION）和 Linux（pthread）
- **专业性**：统一的命名规范、完整的 API 文档、详尽的测试覆盖

## 3. 通用设计模式

### 3.1 泛型实现模式

所有数据结构采用相同的泛型策略：

```c
// 创建：指定元素大小，内部用 memcpy 拷贝
ds_darray_t* arr = ds_darray_create(sizeof(int), 0);

// 使用：传指针
int val = 42;
ds_darray_push_back(arr, &val);

// 读取：传入输出缓冲区
ds_darray_get(arr, 0, &val);
```

关键点：
- `elem_size` 在创建时固定，所有数据拷贝使用 `memcpy`
- 用户负责分配输出缓冲区
- 不存储类型信息，不做类型检查

### 3.2 错误处理约定

所有函数遵循统一的错误返回约定：

| 返回值 | 含义 |
|--------|------|
| `0` | 操作成功 |
| `-1` | 操作失败（参见具体函数文档） |
| `NULL` | 创建/分配失败 |
| `size_t` 返回值 | 0 表示空或无意义（无错误码信息） |

NULL 指针传入视为非法参数，返回 `-1` 或 `NULL`。

### 3.3 内存所有权

- 创建函数返回的对象由调用者拥有，必须调用对应的 `destroy` 释放
- 数据通过 `memcpy` 拷贝进入容器，容器内部数据的生命周期由容器管理
- `destroy` 释放所有内部内存，但不释放用户数据中的指针成员（用户需通过 `free_fn` 回调处理）

### 3.4 命名规范

```
ds_<结构>_<操作>       # 公共 API 函数
ds_<结构>_t             # 不透明类型
ds_<结构>_iter_t        # 迭代器类型
ds_<结构>_fn            # 回调函数类型
<前缀>_<描述>           # 内部函数（static）
```

## 4. 各数据结构设计

### 4.1 动态数组 (darray)

**结构设计：**
```c
struct ds_darray {
    void*  data;       // 连续内存块
    size_t elem_size;  // 单个元素大小
    size_t size;       // 当前元素数
    size_t capacity;   // 当前容量
};
```

**扩容策略：**
- 初始容量：16（可自定义）
- 扩容因子：1.5 倍（比 2 倍更节省内存，且均摊复杂度仍为 O(1)）
- 每次 `push_back` 检查 `size >= capacity` 时自动扩容

**复杂度：**
| 操作 | 时间复杂度 |
|------|-----------|
| 随机访问 `get/set` | O(1) |
| 尾部插入 `push_back` | 均摊 O(1) |
| 中间插入 `insert` | O(n) |
| 尾部删除 `pop_back` | O(1) |
| 中间删除 `erase` | O(n) |

### 4.2 双向链表 (dlist)

**结构设计：**
```c
// 哨兵节点环形结构
// 空链表: sentinel->next == sentinel->prev == sentinel
// 非空:  sentinel->next = 第一个节点, sentinel->prev = 最后一个节点
// 最后一个节点的 next = sentinel, 第一个节点的 prev = sentinel
struct ds_dlist {
    ds_dlist_node_t* sentinel;
    size_t elem_size;
    size_t size;
    ds_dlist_free_fn free_fn;   // 元素析构回调
};
```

哨兵节点设计消除了所有边界情况判断：
- 在第一个节点前插入 = 在哨兵之后插入
- 在最后一个节点后插入 = 在哨兵之前插入
- 空链表时哨兵的前后指针都指向自身

**迭代器有效性：**
迭代器在插入操作后保持有效，在删除操作后指向被删除节点的下一个节点（类似 C++ `std::list`）。

### 4.3 消息队列 (mqueue)

**结构设计：**
```c
struct ds_mqueue {
    unsigned char* buffer;    // 环形缓冲区（预分配）
    size_t max_msgs;          // 最大消息数
    size_t msg_size;          // 每条消息大小
    volatile size_t head;     // 读指针
    volatile size_t tail;     // 写指针
    volatile size_t count;    // 当前消息数
    CRITICAL_SECTION lock;    // 互斥锁
    CONDITION_VARIABLE not_full;   // 生产者等待
    CONDITION_VARIABLE not_empty;  // 消费者等待
    volatile int destroyed;   // 销毁标志
};
```

**线程安全实现：**
- 使用 Windows `CRITICAL_SECTION`（轻量级互斥体，比 Mutex 更快）
- 使用 `CONDITION_VARIABLE`（Windows Vista+，用户态条件变量）
- 所有公共方法在进入时加锁，退出时解锁
- 支持超时唤醒（`SleepConditionVariableCS` 毫秒级超时）

**消息优先级：**
`send_urgent` 支持插队到队首，适用于紧急控制消息。

### 4.4 哈希表 (htable)

**结构设计：**
```c
struct ds_htable {
    htable_entry_t** buckets;   // 桶数组（链地址法）
    size_t bucket_count;        // 桶数
    size_t size;                // 元素个数
    size_t key_size, value_size;
    ds_hash_fn hash_fn;         // 哈希函数
    ds_eq_fn eq_fn;             // 相等比较函数
};
```

**冲突解决：** 链地址法（每个桶挂一个单向链表）

**自动 rehash：**
- 负载因子阈值：0.75
- 扩容倍数：2 倍
- 触发条件：每次 `insert` 后检查
- rehash 时重新计算所有键的桶位置

**内置哈希函数：**
| 函数 | 算法 | 用途 |
|------|------|------|
| `ds_hash_int` | Thomas Wang 整数哈希 | int 键 |
| `ds_hash_str` | DJB2 字符串哈希 | 字符串键 |
| 默认（用户未指定） | FNV-1a 逐字节哈希 | 任意类型键 |

### 4.5 AVL 树 (btree)

**结构设计：**
```c
struct ds_btree_node {
    ds_btree_node_t* left;    // 左孩子
    ds_btree_node_t* right;   // 右孩子
    int height;               // 子树高度（叶子为 1）
    // 键 + 值连续存储
};
```

**平衡算法：**
- 平衡因子 = left.height - right.height
- 允许范围：[-1, 0, 1]
- 超出范围时根据 4 种情况旋转：

| 场景 | 平衡因子 | 操作 |
|------|---------|------|
| LL（左左） | > 1，插入在左子树的左子树 | 右旋 |
| RR（右右） | < -1，插入在右子树的右子树 | 左旋 |
| LR（左右） | > 1，插入在左子树的右子树 | 先左旋再右旋 |
| RL（右左） | < -1，插入在右子树的左子树 | 先右旋再左旋 |

**删除算法：**
- 叶节点：直接删除
- 单子树：用子树替换
- 双子树：用中序后继替换键值，然后递归删除后继节点

### 4.6 图 (graph)

**结构设计：**
```c
struct ds_graph {
    vertex_t* vertices;       // 顶点数组（动态扩容）
    size_t vertex_count;      // 顶点数
    size_t vertex_capacity;   // 容量
    size_t edge_count;        // 总边数
    int directed;             // 是否有向
    int* id_to_index;         // ID 到索引的映射（加速）
};
```

**存储方式：** 邻接表
- 每个顶点维护一个邻接边动态数组
- 适合稀疏图（边数远小于 n²）

**已实现算法：**

| 算法 | 时间复杂度 | 说明 |
|------|-----------|------|
| BFS | O(V + E) | 队列实现，求最短路径（无权） |
| DFS | O(V + E) | 递归实现 |
| Dijkstra | O(V²) | 数组实现（适合中等规模），支持非负权 |
| 拓扑排序 | O(V + E) | Kahn 算法（入度表 + 队列），检测环 |

## 5. 编译与使用

### 5.1 Windows 环境

**方式一：MinGW-w64**
```bash
# 编译所有源文件为目标文件
gcc -std=c99 -Wall -Wextra -O2 -Iinclude -c src/darray.c -o darray.o
gcc -std=c99 -Wall -Wextra -O2 -Iinclude -c src/dlist.c -o dlist.o
gcc -std=c99 -Wall -Wextra -O2 -Iinclude -c src/mqueue.c -o mqueue.o
gcc -std=c99 -Wall -Wextra -O2 -Iinclude -c src/htable.c -o htable.o
gcc -std=c99 -Wall -Wextra -O2 -Iinclude -c src/btree.c -o btree.o
gcc -std=c99 -Wall -Wextra -O2 -Iinclude -c src/graph.c -o graph.o

# 打包为静态库
ar rcs libds.a *.o

# 编译测试
gcc -std=c99 -Wall -Wextra -O2 -Iinclude tests/run_tests.c tests/test_*.c src/*.c -o run_tests.exe -lwinmm
```

**方式二：Visual Studio**
```bash
# 在开发者命令提示符中
cl /Iinclude /Fe:test_runner.exe src/*.c tests/run_tests.c tests/test_*.c
```

**方式三：CMake（跨平台）**
```cmake
cmake_minimum_required(VERSION 3.10)
project(datastruct C)
set(CMAKE_C_STANDARD 99)
add_library(ds src/darray.c src/dlist.c src/mqueue.c src/htable.c src/btree.c src/graph.c)
target_include_directories(ds PUBLIC include)
```

### 5.2 在自己的项目中使用

```c
#include "darray.h"
#include "htable.h"
#include "btree.h"

int main(void) {
    // 动态数组示例
    ds_darray_t* arr = ds_darray_create(sizeof(int), 0);
    for (int i = 0; i < 100; i++) {
        ds_darray_push_back(arr, &i);
    }
    int val;
    ds_darray_get(arr, 42, &val);  // val == 42
    ds_darray_destroy(arr);

    // 哈希表示例
    ds_htable_t* ht = ds_htable_create(sizeof(int), sizeof(double), 0,
                                        ds_hash_int, ds_eq_int);
    int key = 1;
    double v = 3.14;
    ds_htable_insert(ht, &key, &v);
    ds_htable_destroy(ht);

    // AVL 树示例
    ds_btree_t* tree = ds_btree_create(sizeof(int), sizeof(int), cmp_int);
    for (int i = 0; i < 1000; i++) {
        ds_btree_insert(tree, &i, &i);
    }
    printf("height = %zu\n", ds_btree_height(tree));  // ≈ 11
    ds_btree_destroy(tree);

    return 0;
}
```

## 7. 扩展指南

### 7.1 添加新的数据结构

1. 在 `include/` 中创建 `.h` 文件，定义不透明类型和 API
2. 在 `src/` 中创建 `.c` 文件，定义内部结构体并实现
3. 遵循统一的命名规范 `ds_<name>_<operation>`
4. 所有错误返回 `-1`，成功返回 `0`
5. 为每个公共函数编写 Doxygen 注释
6. 在 `tests/` 中创建对应的测试文件
7. 在 `Makefile` 中添加新的源文件

### 7.2 跨平台消息队列

当前 `mqueue.c` 使用 Windows API。如需移植到 Linux，只需替换：
- `CRITICAL_SECTION` → `pthread_mutex_t`
- `CONDITION_VARIABLE` → `pthread_cond_t`
- `SleepConditionVariableCS` → `pthread_cond_timedwait`

## 8. 性能特征

| 数据结构 | 插入 | 删除 | 查找 | 遍历 | 额外内存 |
|----------|------|------|------|------|---------|
| 动态数组 | 均摊 O(1) 尾部 | O(1) 尾部 | O(n) | O(n) | O(capacity - size) |
| 双向链表 | O(1) 头尾 | O(1) 头尾 | O(n) | O(n) | O(n) 指针开销 |
| 消息队列 | O(1) | O(1) | - | - | 固定容量 |
| 哈希表 | 均摊 O(1) | 均摊 O(1) | 均摊 O(1) | O(n + buckets) | O(buckets) |
| AVL 树 | O(log n) | O(log n) | O(log n) | O(n) | O(n) 指针 + 高度 |
| 图 | O(1) 顶点 | O(E) | O(E) | O(V+E) | O(V+E) |
